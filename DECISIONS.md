# Decisions

Decisions made during implementation, and open ones for you to weigh in on.

## Settled during planning (your answers)
- **Full graph extraction** in MVP: deterministic tags/wiki-links + LLM entity extraction with embedding-based dedupe.
- **Hybrid retrieval**: graph-traversal tools + pgvector semantic search, both exposed as AI-SDK tools.
- **Single-user private workspaces**: one workspace per account, hard `userId` isolation everywhere.
- **Background-job ingestion**: durable multi-step jobs.

## Made during implementation (reasonable defaults — flag if you disagree)

0. **Vercel AI Gateway + AI SDK v7** (changed from the initial direct-provider build at your request). All model access routes through the Gateway via `provider/model` slugs — one `AI_GATEWAY_API_KEY` locally, OIDC (no key) in production. This dropped the separate `@ai-sdk/anthropic`/`@ai-sdk/openai` packages and `ANTHROPIC_API_KEY`/`OPENAI_API_KEY`. Required upgrading `ai` v4 → v7 and `@ai-sdk/react` (new `useChat`/transport API, message-parts/metadata). The chat model is `anthropic/claude-opus-4-8`; embeddings `openai/text-embedding-3-small` — both still env-configurable.
1. **Inngest** for background jobs (over QStash / Vercel Queues). Durable steps + debounce fit `parse→chunk→embed→extract` well. Swappable behind `@lattice/ingest`.
2. **Tailwind v3 + shared JS preset** (over Tailwind v4 CSS-first). The plan calls for a shared "Tailwind preset" consumed by `apps/web` and `packages/ui`; v3 `presets` is the well-trodden path. Tokens live once in `@lattice/config/tokens.ts` + mirrored in `globals.css`.
3. **In-house UI components** (over full shadcn/ui + Radix). A small, dependency-light component set (`Button`, `Input`, `Card`, `Badge`, …) styled to the tokens. Fewer deps, full control. Adopt shadcn later if you want richer primitives (dialogs, popovers).
4. **CodeMirror 6** (via `@uiw/react-codemirror`) over Monaco — lighter, better for prose+markdown decorations.
5. **BetterAuth tables co-located in `@lattice/db`** (`auth-schema.ts`) rather than generated by `@better-auth/cli`. One `drizzle-kit migrate` covers auth + app tables, and the schema is migrate-able offline. Trade-off: if you upgrade BetterAuth and its expected schema changes, reconcile `auth-schema.ts` (run the CLI to diff).
6. **Unified `node` table** (document/tag/entity) so the graph view and chat tools query one place; `entity` holds the canonical record, `node(type=entity)` its graph presence.
7. **Entity↔entity `related` edges are upserted, not replaced per-document.** Relations span documents, so a per-doc replace would clobber. Trade-off: removed relations linger until cleanup (see KNOWN_ISSUES).
8. **Document "ready" status = embeddings done.** Entity extraction continues briefly after; the doc is searchable as soon as chunks exist. Status doesn't distinguish "extracting entities."
9. **Embedding dimension pinned to 1536** in the DB schema (`EMBEDDING_DIM`), matching `text-embedding-3-small`. Changing models of a different dimension requires a migration.
10. **Entity merge threshold = 0.86 cosine** (`ENTITY_MERGE_THRESHOLD`), conservative — prefers false splits over false merges. Tune with real data.
11. **`next lint` script removed** to avoid interactive first-run setup; ESLint config is a documented next step.

## Open decisions for you

- **Entity resolution threshold (0.86)** — needs tuning against your real notes. Consider a "merge candidates" review UI rather than auto-merging borderline cases (parked in `.plan/10`).
- **Notes are embedded too** (not just uploads), so semantic search covers authored notes. Slightly more embedding cost for better recall — keep or restrict to uploads?
- **Autosave re-ingest cadence** — every note save enqueues re-embed/re-extract (coalesced by a 20s Inngest debounce). If embedding cost matters, add content-hashing to skip unchanged chunks.
- **Chat model** defaults to `claude-opus-4-8`. Consider a cheaper model (e.g. Sonnet) for cost, since tool-calling does the heavy lifting.
- **Teams/sharing** is explicitly out of MVP. The schema's `userId`-everywhere makes adding a `workspace_id` layer later mechanical but non-trivial.
- **ESLint/Prettier config** — Prettier config is present; ESLint is not wired. Pick a config (e.g. `eslint-config-next` + a shared base) if you want lint in CI.
